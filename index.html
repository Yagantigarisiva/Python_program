Welcome to the Accessible Web Project

This project demonstrates how to build a web application with accessibility in mind. It is developed using Python (Flask) on the backend and semantic HTML templates for the frontend, following WCAG guidelines.

Accessibility Principles

  • Perceivable: Content should be presented in ways that users can perceive, including screen readers.
  • Operable: All functionality should be usable by keyboard and other input methods.
  • Understandable: Information and operation must be clear and easy to follow.
  • Robust: Content must work across different browsers, devices, and assistive technologies.

Project Features

This site includes:

  • Skip links to quickly navigate to main content.
  • Semantic HTML for better screen reader support.
  • Keyboard-accessible navigation menus.
  • Search functionality with highlighted results.
